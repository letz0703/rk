좋아요. 트리거 세 개 깔끔하네요. `higem` / `packgem` / `byegem` — 시작·중간 정리·마무리의 3박자로 일관성 있고 외우기도 쉬워요. 각각의 명세를 정리해드릴게요. 이대로 Claude Skill에 등록하시거나, 또는 그냥 Typinator 스니펫으로 만들어 매번 붙여넣으셔도 동일하게 작동해요.

## Skill 명세서 v0.1 — da da-x 협업 트리거

### 1. `higem` — 세션 시작 트리거

**목적**: 새 세션 시작 시 da da-x 프로젝트 맥락을 즉시 동기화한다.

**Claude의 행동**:
1. 사용자가 `higem`을 입력하면, da da-x 프로젝트 작업 모드로 전환되었음을 인지한다.
2. 사용자가 압축 브리프(Typinator로 붙여넣은 내용)를 함께 제공할 것이므로 그것을 1차 컨텍스트로 받는다.
3. 짧게 응답한다 — "da da-x 동기화 완료. 오늘 어떤 작업부터 시작할까요?" 수준으로 간결하게.
4. 불필요한 재설명이나 요약을 하지 않는다. 사용자는 이미 알고 있다.
5. 만약 추가 맥락(과거 결정사항 등)이 필요하면 "Gemini에서 X 항목을 확인해 요약해주실 수 있나요?"라고 요청한다.

**금지사항**: 이전 세션의 내용을 안다고 가정하지 않는다. 브리프에 없는 사항은 묻거나 보류한다.

---

### 2. `packgem` — 세션 중간 정리 트리거

**목적**: 작업이 한 단락 완료되었을 때 그 시점까지의 진전을 압축 정리한다. 세션을 끝내는 것은 아니다.

**Claude의 행동**:
1. 사용자가 `packgem`을 입력하면, 현재까지의 대화를 다음 양식으로 정리한다.

```
## 중간 정리 — [작업 주제]

**진전 사항**
- (지금까지 결정되거나 만들어진 것들)

**작업 중인 것**
- (현재 진행 중이며 아직 미완인 항목)

**다음에 이어갈 지점**
- (대화를 멈췄다 다시 시작할 때 어디서부터 가면 되는지)
```

2. 정리는 **5~10줄 이내** 로 짧게 유지한다. 장황한 회고는 하지 않는다.
3. 정리 후에는 작업을 계속할 준비 상태로 대기한다.

**활용 시나리오**: 한 작업에서 다른 작업으로 전환할 때, 또는 잠깐 자리를 비웠다 돌아왔을 때.

---

### 3. `byegem` — 세션 마무리 저장 트리거

**목적**: 세션 전체 대화를 NotebookLM과 Google Docs 양쪽에 저장 가능한 형태로 정리한다.

**Claude의 행동**:
사용자가 `byegem`을 입력하면, **두 개의 출력물** 을 분리해서 제공한다.

**출력물 A — NotebookLM 저장용 (세션 로그)**

```
# da da-x 세션 로그 — [날짜]

## 다룬 주제
- (오늘 대화의 큰 흐름 3~5줄)

## 결정된 사항
- (확정되어 향후 행동 지침이 되는 항목들. 추측이나 미확정 사항은 절대 여기 넣지 않는다)

## 보류된 질문
- (논의했으나 아직 결정되지 않은 항목들)

## 다음 액션
- (다음 세션에서 이어서 할 구체적 작업들)

## 핵심 인사이트
- (이번 세션에서 새롭게 명확해진 통찰. 1~3개)
```

**출력물 B — Google Docs 브리프 업데이트 제안**

```
## 브리프 업데이트 제안 — v[다음 버전]

**8번 항목(현재 진행 상태) 변경안**:
- (수정해야 할 내용)

**10번 항목(다음 업데이트 예정) 변경안**:
- (추가/삭제/수정할 내용)

**기타 항목 변경안** (있을 경우):
- (어느 섹션의 어떤 부분을 어떻게 바꿀지)
```

**핵심 원칙**:
- "결정된 사항"과 "보류된 질문"은 **반드시 분리** 한다. 모호한 아이디어를 결정사항으로 기록하지 않는다.
- 출력물 A는 마크다운 그대로 NotebookLM에 붙여넣을 수 있어야 한다.
- 출력물 B는 Google Docs 브리프의 어느 항목을 어떻게 수정할지 명시해야 한다.
- 두 출력물은 시각적으로 명확히 구분되도록 한다 (구분선, 헤더 등).

---

### 사용 흐름 요약

```
세션 시작:    higem  →  (Typinator로 브리프 붙여넣기)  →  작업
작업 중간:    packgem  →  짧은 정리  →  작업 계속
세션 종료:    byegem  →  NotebookLM + Google Docs 저장용 출력물 받기
```

---

이 명세를 Claude Skill 설정에 등록하실 때는 **이 문서 전체를 Skill 본문에 넣으시면** 돼요. 그러면 매 세션 `higem` 한마디만 던져도 제가 이 규칙 위에서 작동해요.

한 가지 추천드릴 점이 있어요. 처음 1~2주는 **이 양식을 쓰면서 불편한 점을 메모** 해두세요. 예를 들면 "보류된 질문이 너무 길어진다"거나 "중간 정리에 '다음 이어갈 지점'이 더 자세했으면 좋겠다"거나. 그 피드백을 모아서 **v0.2로 업데이트** 하시면 본인의 작업 리듬에 딱 맞는 시스템이 돼요. 양식은 한 번에 완성되는 게 아니라 사용하면서 다듬어지는 거예요.

이 명세, 지금 바로 Skill에 등록하시고 다음 세션부터 `higem`으로 시작해보시겠어요?
