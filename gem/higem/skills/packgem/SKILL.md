# packgem — 세션 중간 정리 트리거 (da da-x 협업)

## 목적

작업 한 단락이 마무리되었을 때 그 시점까지의 진전을 압축 정리한다. **세션을 종료하는 것이 아니다.** 잠깐 자리를 비우거나 다른 작업으로 전환할 때 "지금까지 어디까지 왔는지" 빠르게 보존하기 위한 체크포인트.

## 발동 조건

- 사용자 메시지가 정확히 `packgem` (대소문자 무관, 구두점 무관)
- 사용자가 명시적으로 "중간 정리해줘", "여기서 packgem" 등 정리 요청

## 출력 양식 (필수)

다음 양식을 정확히 따른다:

```
## 중간 정리 — [작업 주제]

**진전 사항**
- (지금까지 결정되거나 만들어진 것들)

**작업 중인 것**
- (현재 진행 중이며 아직 미완인 항목)

**다음에 이어갈 지점**
- (대화를 멈췄다 다시 시작할 때 어디서부터 가면 되는지)
```

## 작성 규칙

1. **분량 제한**: 전체 정리는 **5~10줄 이내**. 한 섹션당 1~3 bullet이 이상적이다. 장황한 회고·감상·평가 금지.

2. **구체성**: "여러 가지를 논의함" 같은 추상적 표현 금지. "8번 항목 v2 초안 확정", "B파일 캐릭터 톤 미정" 같은 구체적 결과만 적는다.

3. **분리 원칙**:
   - **진전 사항**: 이미 확정·완료된 것만. 미확정은 절대 여기 오지 않는다.
   - **작업 중인 것**: 진행 중이지만 미완료. 막힌 지점이 있으면 짧게 명시.
   - **다음에 이어갈 지점**: "여기서 멈췄으니 다음엔 X부터" — 재개 지점만 적는다.

4. **작업 주제 제목**: `## 중간 정리 — [작업 주제]`에서 [작업 주제]는 이번 단락의 핵심 작업을 한 구절로. 예: "프롬프트 v2 튜닝", "캐릭터 톤 결정".

5. **정리 후 대기**: 출력만 내고 끝낸다. 다음 행동을 추측해서 제안하지 않는다. 사장님이 다음 지시를 내릴 때까지 대기.

## 응답 예시

**예시 1 — 프롬프트 작업 중간 정리**
```
사용자: packgem

Claude:
## 중간 정리 — 프롬프트 v2 튜닝

**진전 사항**
- 8번 항목 v2 초안 확정 (조명/렌즈 키워드 보강)
- "soft glow" → "morning haze"로 톤 변경 합의

**작업 중인 것**
- 9번 항목 초안 작성 중 — 의상 묘사 부분 막힘

**다음에 이어갈 지점**
- 9번 의상 키워드, 촬영팀 보케팀에 다시 자문 받고 재작성
```

**예시 2 — 짧은 단락**
```
사용자: packgem

Claude:
## 중간 정리 — DA 업로드 카피

**진전 사항**
- MINJU 신작 3장 캡션 확정

**작업 중인 것**
- 태그 셋 — Patreon 호환 검토 중

**다음에 이어갈 지점**
- 태그 결정 후 바로 업로드
```

## 절대 금지사항

- **세션 종료 처리 금지.** packgem은 마무리가 아니라 체크포인트다. byegem과 혼동하지 않는다.
- **"수고하셨습니다", "오늘도 좋은 작업..." 같은 마무리 멘트 금지.**
- **요약 양식 임의 변경 금지.** 위 세 섹션(진전 사항 / 작업 중인 것 / 다음에 이어갈 지점)을 빼거나 바꾸지 않는다.
- **추측 기반 항목 금지.** 대화에서 명시적으로 다뤄지지 않은 항목은 적지 않는다.

## 후속 흐름

packgem 출력 후 사장님이 다음 지시를 내릴 때까지 대기한다. 작업이 계속되거나, 다른 단락으로 넘어가거나, 세션을 종료하면 byegem이 발동된다.
