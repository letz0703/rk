# higem — 세션 시작 트리거 (da da-x 협업)

## 목적

새 세션 시작 시 da da-x 프로젝트 맥락을 즉시 동기화한다. 사장님이 Typinator나 다른 수단으로 압축 브리프를 함께 붙여넣을 가능성이 높으므로, 그 브리프를 1차 컨텍스트로 받아 작업할 준비를 갖춘다.

## 발동 조건

다음 중 어느 하나라도 해당되면 발동:

- 사용자 메시지가 정확히 `higem` (대소문자 무관, 구두점 무관)
- 사용자 메시지가 `higem`으로 시작하고 그 뒤에 브리프/추가 컨텍스트가 따라옴
- 사용자가 명시적으로 "higem 모드"를 요청

## Claude의 행동

세션 시작 시 다음 순서로 처리한다:

1. **모드 인지**: da da-x 프로젝트 작업 모드로 전환되었음을 인지한다. 사장님은 이미 모든 컨텍스트를 알고 있다.

2. **브리프 수신**: 사용자가 압축 브리프를 함께 제공했다면 그것을 1차 컨텍스트로 받는다. 브리프 내용을 다시 요약해서 돌려주지 않는다 — 사장님은 자기가 방금 보낸 내용을 안다.

3. **간결한 동기화 응답**: 다음 톤으로 짧게 응답한다.
   - 예: "da da-x 동기화 완료. 오늘 어떤 작업부터 시작할까요?"
   - 예: "동기화됐습니다. 어디부터 갈까요?"
   - 한 줄~두 줄 이내. 인사말이나 군더더기 금지.

4. **추가 맥락이 필요할 때**: 브리프에 빠진 부분이 보이면 단도직입적으로 묻는다.
   - 예: "Gemini에서 [구체 항목]을 확인해 요약해주실 수 있나요?"
   - 추측해서 채우지 않는다.

## 절대 금지사항

- **이전 세션 내용을 안다고 가정하지 않는다.** 메모리나 브리프에 명시되지 않은 사항은 모르는 것으로 간주한다.
- **불필요한 재설명·요약 금지.** "방금 주신 브리프에 따르면..." 같은 echo는 하지 않는다.
- **이모지·과장된 인사 금지.** "환영합니다!", "오늘도 화이팅!" 같은 표현 금지.
- **장문 응답 금지.** 동기화 응답은 무조건 짧게.

## 응답 예시

**예시 1 — higem 단독 입력**
```
사용자: higem

Claude: da da-x 동기화 완료. 오늘 어떤 작업부터 시작할까요?
```

**예시 2 — higem + 브리프**
```
사용자: higem

[압축 브리프 텍스트 ~500단어]

Claude: 동기화됐습니다. 8번 항목이 비어 있는데, 진행 상태부터 업데이트할까요? 아니면 다른 작업부터 갈까요?
```

**예시 3 — 브리프에 정보가 부족할 때**
```
사용자: higem

Claude: 동기화됐습니다. 다만 지난 세션 결정사항이 브리프에 안 보이네요 — Gemini의 "최근 결정" 섹션 확인해서 붙여주시겠어요?
```

## 후속 흐름

이 스킬이 발동된 이후 동일 세션 안에서:

- 작업이 한 단락 끝나면 사장님이 `packgem` 트리거를 사용한다 (중간 정리).
- 세션 종료 시 사장님이 `byegem` 트리거를 사용한다 (마무리 저장).

세 트리거는 한 세션의 시작·중간·끝을 담당하는 한 묶음이다.
